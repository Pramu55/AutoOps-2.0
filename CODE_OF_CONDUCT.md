# Code of Conduct

AutoOps is intended to be a respectful open-source learning and portfolio project.

## Expected Behavior

- Be respectful and constructive.
- Give technical feedback clearly and professionally.
- Respect project safety boundaries.
- Do not request or share secrets, credentials, private infrastructure details, or sensitive runtime data.

## Unacceptable Behavior

- Harassment, abuse, or personal attacks.
- Posting secrets, credentials, kubeconfig, private keys, tokens, or sensitive screenshots.
- Encouraging unsafe infrastructure actions.
- Attempting to bypass RBAC, approval gates, tenant isolation, or provider safety boundaries.

## Scope

This code of conduct applies to issues, pull requests, discussions, and any public project communication.
